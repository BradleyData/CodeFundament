formatting

Code

Code represents text as blocks of code.

A kbd indicates text to be entered by the user.

An output displays the results of a calculation; generally from client-side JavaScript.

15% A progress represents the amount of a task completed. It should be used with JavaScript and should never be used to replace a meter.

A progress with no value assigned.

A samp specifies text as sample code.

A var is used to indicate the text is an instance of a variable.

Reference

An abbr is an abbreviation.

The address is used to indicate the contact details for the author or owner of the document.

A block quotation (also known as a long quotation or extract) is a quotation in a written document that is set off from the main text as a paragraph, or block of text, and typically distinguished visually using indentation and a different typeface or smaller size quotation.

A cite indicates a citation or reference to another source.

Del marks text as deleted.

A dfn is a phrase being defined.

Ins marks text as inserted.

A mark is used to highlight a section of text for reference purposes.

A q is a short inline quotation.

Stylistic

A b is stylistically different text from normal text, without any special importance.

An em emphasizes text.

An i is italicized text without any special importance or meaning.

    A pre contains
    preformatted text   and maintains      whitespace.

A strong is used for strongly emphasized text and indicates importance.

Sub is used for subscripted text as in H2O.

Sup is used for superscripted text as in x2.

The wbr is used to indicate a WordBreakOpportunityWhereTheBrowserMayOptionallyBreakTheLineIfNecessary.

Text Direction

A bdi represents text that is isolated from other text for purposes of formatting in a different direction.
